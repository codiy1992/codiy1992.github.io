01-管理AWS云平台资源的几种方式

codiy 2020-04-20 10:47:08

与 AWS 云平台交互，主要有四种方式：控制台、命令行工具、SDK、CloudFormtaion，本篇文章探讨这四种方式管理AWS资源的优缺点，以及使用何种方式能最舒服的管理 AWS 资源

思考一个问题: 管理 AWS 资源，我们想要什么特性?

#1、使用AWS控制台

使用控制台是与 AWS 进行交互最直观最方便的一种方式，只需要拥有aws控制台的相关账号密码，即可登录控制台进行操作相关操作.

缺点: 无法自动化、具有一定规模复杂度时显得很笨拙，需要很多人工成本。

#2、使用命令行工具awscli

因为 AWS 每个服务都具有相当程度复杂的参数定义，而在使用命令行工具 awscli 时，每次需要构造复杂的参数定义，需要编写许多辅助的 shell 脚本才能比较好的使用它. 并且操作是无状态的, 需要自行编写回滚脚本.

缺点: 参数定义复杂、无法自动回滚

#3、使用语言相关的SDK或调用API

AWS 的每一种服务都具有相应的 API 调用接口, 并为各种语言提供了相关的 SDK 支持(Java SDK, Python SDK…)

通常 SDK 仅当业务需要时接入业务应用代码中使用，要使用 SDK 来全面管理 AWS 资源也不是方便的事情。

#4、使用CloudFormation

CloudFormation是四种方式中最推荐的一种。

当然，我们仍然无法避开那些复杂的资源参数定义，不过好在这些参数以 yaml 或 json 的形式组织，并且有完善属性参数定义文档，这个文档比命令行工具 awscli 的文档好太多了，awscli 的文档有时讲的不是很清楚。

优点:

  1. 明确清晰的属性参数定义
  2. 可重复使用的模板定义
  3. 提供堆栈资源输出与引用
  4. 可方便的进行资源回滚

然而如果直接使用 Cloudformation 的模板来定义你需要的 AWS 资源，仍然十分的笨拙。好在已有一些成熟的工具可以帮助我们快速的生成 Cloudformation 资源模板，并能很好的复用一些已有的模板.

Cloudformation管理工具:

  1. 使用 ansible 的 cloudformation 模块和 ansible 内置的 jinja2 模板引擎
  2. 使用 AWS CDK 以编程的方式生成并部署相应的 cloudformation 模板
